Here is a comparison between the new `R4` script (`r4_macro_agent.py`) and the previous four-script architecture (`generate_report.py`, `fedwatch.py`, `finviz_futures.py`, `treasury_yields.py`).

### Where R4 Has Improved

* **Stability vs. Brittle Scraping**: The old architecture relied heavily on brittle web scraping, using BeautifulSoup for HTML tables and Selenium/headless browsers to extract data from JS-rendered widgets (like CME FedWatch). R4 eliminates scraping entirely, replacing it with official public APIs (BLS API), structured calendar feeds (ICS/RSS for Fed and BLS schedules), and robust financial libraries (`yfinance`).


* **Data Processing with Pandas**: Instead of manually parsing text and floats from HTML tables, R4 leverages the `pandas` library. This allows for highly accurate time-series data manipulation, such as calculating 5-day rolling returns and 20-day Exponential Moving Averages (EMA20) flawlessly.


* **Analytical Depth \u0026 Scoring**: The old scripts provided a "mechanical readout" without passing judgment. R4 introduces a quantitative scoring engine via a `Signal` dataclass and a `threshold_score` function. It automatically scores metrics as bullish (+1), bearish (-1), or neutral (0) based on predefined thresholds.


* **Sector Rotation Analysis**: R4 introduces a completely new analytical layer by tracking S&P 500 sector proxies (e.g., Technology, Health Care) and calculating the performance spread between "Cyclical" and "Defensive" sectors to determine market risk appetite.


* **Unified Architecture**: It consolidates multiple fragmented files into a single, clean, object-oriented script that is easier to run and maintain.



### Output Results

The old architecture generated a single Markdown file (`fed_market_report_YYYY-MM-DD.md`).

The new R4 agent creates a comprehensive, timestamped output directory (defaulting to `output/agents`) containing four distinct files:

1. **Markdown Report**: `macro_agent_output_W{week}.md`.


2. **Macro Data CSV**: `r4_macro_market_data_W{week}.csv` (structured data for macro signals).


3. **Sector Data CSV**: `r4_sector_rotation_W{week}.csv` (structured data for sector performance).


4. **Visual Evidence PNG**: `r4_macro_evidence_W{week}.png` (an automatically generated data visualization chart).
